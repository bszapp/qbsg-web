# 签变时光 前端

基于 Vue 3 + Vite 构建的固件激活码服务前端，对接后端 REST API，支持用户注册登录、积分充值、固件浏览与激活码获取等功能。

---

## 功能概览

### 导航页面

| 路径 | 页面 | 是否需要登录 |
|------|------|------------|
| `/` | 主页（施工中占位） | 否 |
| `/shop` | 资源（施工中占位） | 否 |
| `/nodes` | 固件库 | 否 |
| `/activation` | 激活码 | **是** |
| `/me` | 我的（账户与充值） | **是** |
| `/me/settings` | 个人设置 | **是** |

---

### 用户认证

- **登录 / 注册弹窗**：点击右上角头像或访问受保护页面时自动弹出。
- **记住我**：登录时勾选后，用户名和密码会保存在 `localStorage`，下次打开自动填入。
- **忘记密码**：提示用户联系管理员凭历史交易记录找回。
- **人机验证（Captcha）**：登录和注册均需完成 Cloudflare Turnstile 验证。
- **关闭弹窗跳转逻辑**：若弹窗由路由鉴权触发，关闭时未完成登录会自动返回上一页；若上一页也需要登录，则直接跳转到主页，避免死循环。

---

### 固件库（`/nodes`）

- 从后端拉取所有可用固件分组及版本列表。
- 每个固件卡片展示：名称、描述、屏幕尺寸、固件 ID、缩略图。
- 点击分组的「获取激活码」按钮，跳转到激活码页面并自动填入提供商和型号。

---

### 激活码（`/activation`）

- 选择固件提供商、固件型号，输入设备 MAC 地址后提交。
- 后端处理 MAC 格式校验，前端不做限制。
- 激活成功后，在页面展示激活码，并通过 Toast 提示本次消耗积分数。
- 重复激活同设备同型号通常不重复扣积分，后端会返回相应标志。
- 当前积分在激活成功后实时更新。

---

### 账户与充值（`/me`）

- 显示当前用户名与剩余积分。
- **快捷面额充值**：后端下发推荐充值档位（含折扣信息），点击自动填入积分数量。
- **自定义积分充值**：输入任意积分数量，实时预估应付金额（支持折扣计算）。
- 充值通过支付宝完成，创建订单后自动跳转支付页面。
- **订单号手动认领**：支付回跳后若未自动到账，可输入商家订单号手动确认。

---

### 个人设置（`/me/settings`）

- 修改登录密码（需输入当前密码 + 新密码确认）。
- 注销账号（二次确认后永久删除账号数据）。

---

### 全局功能

- **亮色 / 暗色主题切换**：点击右上角图标切换，偏好保存在 `localStorage`，首次访问根据系统设置自动适配。
- **Toast 通知**：全局轻提示，支持 `success / error / warning / info` 四种类型，自动消失。
- **Confetti 彩带动画**：特定操作后触发全屏庆祝动画（目前主页有彩蛋按钮）。
- **NProgress 进度条**：路由切换时顶部显示加载进度条。
- **路由鉴权**：需要登录的页面在未登录状态下显示提示卡片，并自动弹出登录弹窗。

## 项目结构

```
src/
├── assets/              # 静态资源
├── components/
│   ├── AppLayout.vue    # 主布局（导航栏、主题切换、登录弹窗）
│   ├── AuthGateModal.vue # 登录 / 注册弹窗
│   ├── CaptchaWidget.vue # Cloudflare Turnstile 人机验证
│   ├── ConfettiExplosion.vue # 单次彩带爆炸动画
│   ├── ConfettiOverlay.vue   # 全屏彩带覆盖层
│   ├── NavIcon.vue      # 导航图标
│   └── ToastNotification.vue # Toast 通知组件
├── composables/
│   ├── useAuth.js       # 认证状态管理（登录、注册、退出、token 持久化）
│   ├── useConfetti.js   # 彩带触发逻辑
│   └── useToast.js      # Toast 通知管理
├── config/
│   └── app.js           # 站点部署路径、API 基础 URL 配置
├── router/
│   └── index.js         # 路由定义（含导航图标与鉴权元数据）
├── services/
│   └── api.js           # HTTP 请求封装（postJson、normalizeMessage）
├── styles/              # 全局样式（变量、布局、页面、基础重置）
├── views/
│   ├── Dashboard.vue    # 主页
│   ├── Shop.vue         # 资源页
│   ├── Nodes.vue        # 固件库
│   ├── Orders.vue       # 激活码
│   ├── Account.vue      # 账户充值
│   └── AccountSettings.vue # 个人设置
├── App.vue              # 根组件
└── main.js              # 应用入口
```

## 配置说明

编辑 `src/config/app.js`：

```js
// 站点部署目录，根目录填 '/'，子目录填如 '/new/'
export const APP_BASE_PATH = '/new/'
```

本地开发时 API 请求自动代理至 `https://qbsg.top`，生产环境使用相对路径。
